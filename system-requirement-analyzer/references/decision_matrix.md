# 系统建设工具决策矩阵

## 决策维度定义

### 1. 复杂度评估
- **简单**：单一功能，无复杂业务逻辑，1-2个数据实体
- **中等**：多模块协作，有业务流程，3-5个数据实体
- **复杂**：多系统集成，复杂算法，5个以上数据实体，高并发

### 2. 数据需求
- **强数据管理**：需要持久化存储、复杂查询、数据关系
- **弱数据管理**：临时数据、简单缓存、无需持久化

### 3. 流程需求
- **强流程**：复杂审批、多分支条件、并行处理
- **弱流程**：简单顺序执行、无审批需求

### 4. AI需求
- **强AI**：需要理解、推理、生成、多模态处理
- **弱AI**：仅需简单问答、信息检索

### 5. 集成需求
- **强集成**：需对接多个外部系统、API
- **弱集成**：独立运行、飞书生态内闭环

### 6. 时效需求
- **快速上线**：1-2周内需要可用版本
- **正常周期**：1-2个月完整建设周期

## 组合方案推荐

### 方案1: aPaaS + 通用智能体
**适用场景**：复杂业务系统 + 智能分析/决策
**分工**：
- aPaaS：数据管理、表单、审批、权限
- 通用智能体：数据分析、报告生成、智能建议

### 方案2: 垂类智能体 + 工作流应用
**适用场景**：智能客服 + 工单流转
**分工**：
- 垂类智能体：用户问答、意图识别
- 工作流应用：工单创建、状态流转、通知

### 方案3: 妙搭 → aPaaS
**适用场景**：需求不确定，需快速验证
**路径**：
- 阶段1：妙搭快速原型验证核心功能
- 阶段2：验证通过后迁移至aPaaS进行企业级改造

### 方案4: 工作流应用 + 通用智能体
**适用场景**：复杂自动化流程 + 智能决策
**分工**：
- 工作流应用：流程编排、系统对接
- 通用智能体：条件判断、内容生成、异常处理

## 反模式警示

### ❌ 不推荐的组合
1. **aPaaS + 垂类智能体**：能力重叠，垂类智能体无法补充aPaaS的短板
2. **妙搭 + 复杂工作流**：妙搭不适合复杂流程编排
3. **纯通用智能体处理强数据需求**：缺乏数据持久化和事务保证

### ⚠️ 需要谨慎的场景
1. **单一工作流应用做CRUD系统**：用户体验差，后期难维护
2. **妙搭承载核心业务**：缺乏企业级保障
3. **垂类智能体处理复杂推理**：超出其能力边界
