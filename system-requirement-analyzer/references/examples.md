# 需求分析案例库

## 案例1：员工请假审批系统

### 原始需求
"我们需要一个员工请假系统，员工可以提交请假申请，主管审批，HR备案，还要能看到请假统计。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 中等 |
| 数据实体 | 员工、请假单、审批记录、假期余额 |
| 流程需求 | 多级审批（主管→HR），条件分支（根据请假类型） |
| 数据管理 | 强需求，需要持久化和查询统计 |
| AI需求 | 弱，主要是规则判断 |
| 集成需求 | 弱，飞书生态内闭环 |

### 推荐方案
**主要工具**：飞书aPaaS平台
**理由**：
- 需要完整的数据模型（员工、请假单、假期余额）
- 需要多级审批流程
- 需要统计报表功能
- aPaaS的表单和流程能力完全匹配

### 建设内容
1. 数据建模：员工表、请假单表、假期余额表、审批记录表
2. 流程设计：提交→主管审批→HR备案→完成
3. 页面设计：员工申请页、审批页面、统计看板
4. 权限配置：员工、主管、HR不同数据权限

## 案例2：智能IT支持助手

### 原始需求
"我们希望有个机器人能回答员工的IT问题，比如密码重置、软件安装、网络故障等，回答不了的问题能自动转工单。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 简单-中等 |
| 数据实体 | 知识库文档、工单 |
| 流程需求 | 问答为主，转工单为简单流程 |
| 数据管理 | 中等，主要是知识库管理 |
| AI需求 | 强，需要理解问题、匹配知识、判断转人工 |
| 集成需求 | 中等，需对接工单系统 |

### 推荐方案
**主要工具**：飞书垂类智能体 + 飞书工作流应用
**理由**：
- 垂类智能体：挂载IT知识库，提供7×24问答服务
- 工作流应用：处理转工单流程，对接现有工单系统

### 建设内容
1. 知识库建设：整理IT常见问题文档
2. 垂类智能体配置：配置系统提示词，挂载知识库
3. 工作流设计：意图识别→匹配知识→置信度判断→转工单/直接回答
4. MCP扩展：如需调用外部系统，配置MCP工具

## 案例3：销售商机管理系统

### 原始需求
"我们需要管理销售商机，从线索跟进到成交，要有客户画像，还要AI辅助分析赢单概率和跟进建议。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 复杂 |
| 数据实体 | 客户、联系人、商机、跟进记录、合同 |
| 流程需求 | 商机阶段流转、审批 |
| 数据管理 | 强需求，复杂关系和数据量 |
| AI需求 | 强，客户画像、赢单分析、跟进建议 |
| 集成需求 | 中等，可能需对接财务/合同系统 |

### 推荐方案
**主要工具**：飞书aPaaS平台 + 飞书aily通用智能体
**理由**：
- aPaaS：承载CRM核心功能（客户、商机、合同管理）
- 通用智能体：AI分析能力（客户画像分析、赢单概率、跟进建议）

### 建设内容
1. aPaaS部分：
   - 数据建模：客户表、商机表、跟进记录表、合同表
   - 页面设计：客户列表、商机看板、详情页面
   - 流程设计：商机阶段流转审批
2. 通用智能体部分：
   - 开发Skill：读取aPaaS数据进行分析
   - 客户画像生成
   - 赢单概率预测
   - 跟进策略建议

## 案例4：跨系统数据同步

### 原始需求
"我们HR系统、财务系统、飞书组织数据需要每天同步，人员入职离职要自动同步到各系统。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 中等 |
| 数据实体 | 人员信息、组织架构 |
| 流程需求 | 强，定时触发+条件处理+错误处理 |
| 数据管理 | 中等，主要是数据传输和日志 |
| AI需求 | 弱，主要是规则匹配 |
| 集成需求 | 强，多系统API对接 |

### 推荐方案
**主要工具**：飞书aily工作流应用
**理由**：
- 工作流应用最适合跨系统集成场景
- 支持定时触发、Webhook触发
- 可视化的错误处理和重试机制
- 可直接暴露API供外部调用

### 建设内容
1. 流程设计：
   - 定时触发器（每天凌晨）
   - 读取飞书组织架构
   - 数据比对和变更检测
   - 调用HR/财务系统API
   - 日志记录和异常通知
2. 异常处理：
   - API调用失败重试
   - 数据不一致告警
   - 人工介入审批节点

## 案例5：快速原型 - 会议室预定助手

### 原始需求
"我们想快速验证一个想法：让员工通过对话方式预定会议室，比如'帮我订个明天下午能坐10人的会议室'。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 简单 |
| 数据实体 | 会议室、预定记录 |
| 流程需求 | 简单查询和预定 |
| 数据管理 | 弱，验证阶段可简化 |
| AI需求 | 中等，自然语言理解 |
| 时效需求 | 快速上线，1周内 |

### 推荐方案
**主要工具**：飞书妙搭
**理由**：
- 快速原型验证场景
- 自然语言交互可快速实现
- 无需复杂数据建模
- 验证成功后可迁移至aPaaS

### 建设内容
1. 通过自然语言描述需求生成原型
2. 配置会议室数据（可mock）
3. 测试核心交互流程
4. 收集反馈，确定是否正式建设

## 案例6：复杂数据分析报告

### 原始需求
"我们需要每周生成一份经营分析报告，整合销售、财务、运营数据，要有洞察发现和可视化图表。"

### 需求拆解
| 维度 | 评估结果 |
|------|---------|
| 复杂度 | 复杂 |
| 数据实体 | 多源数据（销售、财务、运营） |
| 流程需求 | 定时执行、多步骤处理 |
| 数据管理 | 中等，主要是数据获取和临时处理 |
| AI需求 | 强，数据洞察、报告生成 |
| 集成需求 | 强，多数据源整合 |

### 推荐方案
**主要工具**：飞书aily工作流应用 + 飞书aily通用智能体
**理由**：
- 工作流应用：定时触发、数据获取、流程编排
- 通用智能体：数据分析、洞察生成、报告撰写

### 建设内容
1. 工作流应用部分：
   - 定时触发（每周一早上）
   - 数据获取节点（连接各业务系统）
   - 数据预处理节点
2. 通用智能体部分：
   - 数据分析Skill
   - 洞察发现
   - 报告生成（飞书文档）
   - 图表生成
