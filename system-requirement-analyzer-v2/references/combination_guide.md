# 组合方案指南

> **使用时机**：当六维度评分触发了多条单工具规则，或命中了组合方案触发规则（C1-C3）时，参考本文件确定组合方案。

---

## 组合方案一览

### 方案1：aPaaS + aily工作流

**触发条件**：数据需求 ≥ 4 **且** AI 需求 ≥ 4

**适用场景**：复杂业务系统 + 智能分析/决策

**分工**：
| 模块 | 承载工具 | 职责 |
|------|---------|------|
| 数据管理 | aPaaS | 数据建模、表单、CRUD 操作 |
| 审批流程 | aPaaS | 审批规则、流程流转 |
| 权限管控 | aPaaS | 数据级/字段级/操作级权限 |
| 智能分析 | 通用智能体 | 数据分析、洞察发现、报告生成 |
| 智能建议 | 通用智能体 | 基于数据的推荐和预测 |

**集成方式**：通用智能体通过 Skill 调用 aPaaS 开放 API 读写数据

**典型案例**：销售商机管理系统（aPaaS 管理客户/商机数据，通用智能体做赢单分析和跟进建议）

---

### 方案2：垂类智能体 + 工作流应用

**触发条件**：AI 需求 ≥ 3 **且** 集成需求 ≥ 3 **且** 流程需求 ≥ 3

**适用场景**：智能客服 + 工单流转

**分工**：
| 模块 | 承载工具 | 职责 |
|------|---------|------|
| 用户问答 | 垂类智能体 | 知识库匹配、意图识别、直接回答 |
| 知识管理 | 垂类智能体 | 挂载知识库文档 |
| 工单流转 | 工作流应用 | 工单创建、状态流转、通知 |
| 系统对接 | 工作流应用 | 调用外部工单/CRM 系统 |

**集成方式**：垂类智能体通过 MCP 工具调用工作流应用暴露的 API

**典型案例**：智能 IT 支持助手（垂类智能体回答常见问题，无法回答时工作流自动创建工单）

---

### 方案3：工作流应用 + 通用智能体

**触发条件**：流程需求 ≥ 3 **且** AI 需求 ≥ 4 **且** 集成需求 ≥ 3

**适用场景**：复杂自动化流程 + 智能决策

**分工**：
| 模块 | 承载工具 | 职责 |
|------|---------|------|
| 流程编排 | 工作流应用 | 定时/事件触发、流程节点执行 |
| 系统对接 | 工作流应用 | HTTP 调用、Webhook、数据同步 |
| 智能判断 | 通用智能体 | 条件判断、内容生成、异常分析 |
| 报告生成 | 通用智能体 | 数据分析、洞察、文档输出 |

**集成方式**：工作流应用在 AI 节点中调用通用智能体的能力

**典型案例**：每周经营分析报告（工作流定时获取多源数据，通用智能体分析生成报告）

---

### 方案4：妙搭 → aPaaS（分阶段迁移）

**触发条件**：时效需求 ≥ 4 **且** 复杂度 = 3（较紧急但中等复杂度）

**适用场景**：需求不确定，需快速验证后正式建设

**阶段规划**：
| 阶段 | 工具 | 目标 | 周期 |
|------|------|------|------|
| 阶段1：验证 | 妙搭 | 核心交互流程验证，收集用户反馈 | 1 周 |
| 阶段2：决策 | — | 根据反馈决定是否正式建设 | 2-3 天 |
| 阶段3：建设 | aPaaS | 企业级改造，补齐权限/审计/性能 | 2-4 周 |

**注意事项**：妙搭原型 **不可** 直接用于生产环境，阶段1仅限内部验证

## 组合方案选择优先级

当多个组合方案都可能适用时，按以下优先级排序：

| 优先级 | 判断条件 | 选择方案 |
|--------|---------|---------|
| 1 | 数据需求是最高分维度（≥4）且 AI 也高 | 方案1（aPaaS + 通用智能体） |
| 2 | AI 需求是最高分维度 且 集成/流程也高 | 看流程是否核心：流程核心选方案3，问答核心选方案2 |
| 3 | 时效紧急（≥4）且需求尚未完全确定 | 方案4（妙搭 → aPaaS） |
| 4 | 各维度分值较均匀，无突出维度 | 方案1（aPaaS + 通用智能体）作为默认组合 |
