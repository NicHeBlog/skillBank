# 六维度评分标准与量化决策规则

> **本文件是工具选择的唯一决策依据。** SKILL.md 的第三步和第四步必须引用本文件。

---

## 一、六维度评分标准（1-5 分）

### 1. 数据需求

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 无持久化需求 | 纯对话交互，不需要存储数据 |
| 2 | 轻量数据 | 1-2个简单数据实体，无复杂关联，可用表格/文档承载 |
| 3 | 中等数据 | 3-5个数据实体，有基础关联关系，需要列表查询和筛选 |
| 4 | 强数据 | 5-10个数据实体，多对多关系，需要复杂查询、统计报表 |
| 5 | 重度数据 | 10+数据实体，复杂关联/层级，大数据量，需要事务保证 |

### 2. 流程需求

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 无流程 | 一次性操作，无需审批或状态流转 |
| 2 | 简单流程 | 单步审批或简单状态流转（如：待办→完成） |
| 3 | 中等流程 | 多级审批（2-3级），有条件分支（如：金额>5000走不同审批） |
| 4 | 复杂流程 | 并行审批、会签、多分支条件、有回退和加签 |
| 5 | 重度流程 | 跨部门多链路、并行+串行混合、动态审批人、SLA 时限控制 |

### 3. AI 需求

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 无 AI | 纯规则逻辑，不涉及自然语言处理或智能判断 |
| 2 | 轻度 AI | 简单关键词匹配、信息检索 |
| 3 | 中度 AI | 需要意图理解、知识库问答、文本分类 |
| 4 | 强 AI | 需要推理、生成、分析总结、多轮对话理解 |
| 5 | 重度 AI | 需要自主规划、多工具协作、多模态处理、复杂推理链 |

### 4. 集成需求

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 无集成 | 独立运行，不对接任何外部系统 |
| 2 | 飞书内闭环 | 仅使用飞书自有能力（文档、消息、日历等） |
| 3 | 轻度集成 | 对接1-2个外部系统，调用频率低 |
| 4 | 中度集成 | 对接3-5个外部系统，有双向数据同步需求 |
| 5 | 重度集成 | 对接5+外部系统，高频调用，需要统一API网关或数据中台 |

### 5. 时效需求

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 不急 | 3个月以上的建设周期可接受 |
| 2 | 正常 | 1-2个月完整建设周期 |
| 3 | 较紧 | 2-4周内需要上线核心功能 |
| 4 | 紧急 | 1-2周内需要可用版本 |
| 5 | 极紧急 | 1周内/几天内需要可用版本 |

### 6. 复杂度（综合评估）

| 分值 | 标准 | 典型特征 |
|------|------|---------|
| 1 | 极简 | 单一功能点，无业务逻辑 |
| 2 | 简单 | 1-2个功能模块，简单业务逻辑 |
| 3 | 中等 | 3-5个功能模块，有业务流程和规则 |
| 4 | 复杂 | 多模块协作，复杂业务规则，多角色 |
| 5 | 极复杂 | 多系统集成、复杂算法、高并发、多租户 |

## 二、量化决策规则表

> **匹配规则**：从 Rule 1 开始逐条检查，满足条件即命中。一个需求可能命中多条规则，命中多条时参考组合方案。

### 单工具规则

| 规则编号 | 条件 | 推荐工具 | 核心理由 |
|---------|------|---------|---------|
| R1 | 数据≥4 **且** 流程≥3 | aPaaS 平台 | 强数据建模+审批流程是 aPaaS 核心优势 |
| R2 | 数据≥4 **且** 流程<3 **且** AI<3 | aPaaS 平台 | 强数据需求但无复杂AI，aPaaS 的数据管理能力最匹配 |
| R3 | AI≥4 **且** 数据<3 **且** 复杂度≥3 | aily 通用智能体 | 强AI+复杂推理，通用智能体的自主规划能力匹配 |
| R4 | AI=3 **且** 数据≤2 **且** 流程≤2 | aily 垂类智能体 | 中度AI+轻数据+轻流程，垂类智能体低成本高效 |
| R5 | 集成≥4 **且** AI<4 **且** 数据<4 | aily 工作流应用 | 强集成场景，工作流的多触发器+HTTP节点最匹配 |
| R6 | 流程≥3 **且** 集成≥3 **且** 数据<3 | aily 工作流应用 | 跨系统流程编排，工作流是最佳选择 |
| R7 | 时效≥4 **且** 复杂度≤2 | 妙搭 | 极紧急+低复杂度，妙搭分钟级生成 |
| R8 | 时效≥4 **且** 复杂度=3 | 妙搭 → aPaaS（分阶段） | 较紧急+中等复杂度，先妙搭验证再迁移 |

### 组合方案触发规则

| 规则编号 | 条件 | 推荐组合 | 参考 |
|---------|------|---------|------|
| C1 | 数据≥4 **且** AI≥4 | aPaaS + 通用智能体 | 见 combination_guide.md 方案1 |
| C2 | AI≥3 **且** 集成≥3 **且** 流程≥3 | 垂类智能体 + 工作流应用 | 见 combination_guide.md 方案2 |
| C3 | 流程≥3 **且** AI≥4 **且** 集成≥3 | 工作流应用 + 通用智能体 | 见 combination_guide.md 方案3 |

### 兜底规则

| 规则编号 | 条件 | 处理方式 |
|---------|------|---------|
| D1 | 无规则命中 | 取各维度最高分对应的工具，置信度标记为"低"，建议用户补充信息 |
| D2 | 多条单工具规则同时命中 | 优先选择条件匹配度更高（满足条件数更多）的规则，并检查是否适合组合方案 |

## 三、工具-场景快速映射表

> 用于辅助校验推荐结果是否合理。

| 典型场景 | 首选工具 | 备选/组合 |
|---------|---------|----------|
| 审批类系统（请假、报销、采购） | aPaaS | — |
| CRM / 客户管理 | aPaaS | aPaaS + 通用智能体 |
| 内部知识库问答 | 垂类智能体 | 通用智能体 |
| 智能客服 | 垂类智能体 | 垂类智能体 + 工作流 |
| 数据同步 / ETL | 工作流应用 | — |
| AI 助手 / Agent | 通用智能体 | 通用智能体 + 工作流 |
| 定时报表生成 | 工作流应用 | 工作流 + aPaaS |
| 快速原型 / MVP | 妙搭 | 妙搭 → aPaaS |
| 复杂分析决策 | 通用智能体 | 通用智能体 + aPaaS |
| 跨系统流程自动化 | 工作流应用 | 工作流 + 通用智能体 |
