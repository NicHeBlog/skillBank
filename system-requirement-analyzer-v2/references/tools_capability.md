# 飞书工具能力边界详解

> **最后更新**：2026-03-30 | **适用版本**：飞书 7.x

---

## 1. 飞书 aPaaS 平台

**一句话定位**：企业级低代码平台，核心优势在数据建模和业务流程。

### 核心能力
- **数据建模**：可视化数据表设计、关联关系配置、字段类型丰富（文本、数字、日期、附件、引用等）
- **业务流程**：流程画布设计、节点配置、条件分支、并行/串行流程
- **审批流程**：多级审批、会签/或签、转交、加签、审批规则引擎
- **FaaS 云函数**：支持 Python/Node.js，可调用外部 API、复杂业务逻辑处理
- **前端页面**：表单页面、列表页面、详情页面、自定义页面（低代码）
- **权限管理**：数据级权限、字段级权限、操作级权限
- **集成能力**：Webhook、API 开放、连接器生态

### 适用场景
- 中等复杂度业务系统（ERP、CRM、HRM 等）
- 需要强数据建模和关系管理的场景
- 复杂审批工作流需求
- 需要自定义业务逻辑计算
- 多角色协同的表单类应用

### 局限性
- 重度定制化 UI 场景受限
- 复杂算法/AI 模型集成需借助 FaaS
- 大规模并发性能有上限

---

## 2. 飞书 aily 通用智能体

**一句话定位**：全能型 AI Agent，核心优势在自主规划和多工具协作。

### 核心能力
- **全知全能**：具备飞书文档、聊天等上下文感知能力
- **灵活扩展**：支持自定义 Skills 扩展能力
- **多模态交互**：文本、图片、文件理解
- **知识融合**：可整合多源知识进行推理
- **自主规划**：复杂任务的自主拆解和执行

### 适用场景
- 知识密集型咨询场景
- 需要上下文理解和推理的复杂任务
- 多步骤、多工具协作的自动化流程
- 需要自然语言交互的智能助手
- 跨系统数据整合分析

### 局限性
- 不擅长结构化数据管理
- 无法替代传统数据库应用
- 审批流程等结构化流程需配合其他工具

---

## 3. 飞书 aily 智能体（垂类）

**一句话定位**：垂直领域问答专家，核心优势在知识库问答和低成本部署。

### 核心能力
- **知识库链接**：可挂载专属知识库文档
- **提示词调优**：支持系统提示词个性化配置
- **MCP 扩展**：支持 MCP 工具调用扩展能力
- **垂直场景**：针对特定领域优化的回复
- **低成本部署**：相对轻量，快速上线

### 适用场景
- 客服问答机器人
- 内部知识库查询助手
- 领域专家咨询（HR 政策、IT 支持等）
- 标准化流程问答
- 轻量级智能问答场景

### 局限性
- 复杂业务流程执行能力有限
- 数据存储和计算能力弱
- 不适合强表单/审批类需求

---

## 4. 飞书 aily 工作流应用

**一句话定位**：自动化流程编排引擎，核心优势在跨系统集成和触发器能力。

### 核心能力
- **Workflow 编排**：可视化流程编排、条件分支、循环控制
- **多触发器**：定时触发、事件触发、手动触发、Webhook 触发
- **API 暴露**：可对外暴露为 API 服务
- **机器人集成**：可作为机器人提供服务
- **多节点类型**：AI 节点、代码节点、飞书节点、HTTP 节点等
- **异步执行**：支持长时间运行任务

### 适用场景
- 自动化工作流（数据同步、消息通知等）
- 跨系统集成场景
- 定时任务 / 批处理任务
- AI 能力编排和组合
- 需要对外暴露服务的场景

### 局限性
- 复杂 UI 界面不支持
- 用户交互体验有限
- 不适合 CRUD 类业务系统

---

## 5. 飞书妙搭（Vibe Coding）

**一句话定位**：AI 驱动的快速原型工具，核心优势在分钟级应用生成。

### 核心能力
- **自然语言生成**：通过对话生成完整应用
- **快速原型**：分钟级应用搭建
- **AI 驱动**：AI 辅助代码生成、UI 设计
- **一键部署**：快速发布和分享
- **迭代优化**：持续对话迭代改进

### 适用场景
- 快速原型验证（MVP）
- 轻量级工具应用
- 个人效率工具
- 简单信息展示类应用
- 创意快速实现

### 局限性
- 复杂业务逻辑支持有限
- 企业级功能（权限、审计）弱
- 大规模使用有性能限制
- 深度定制化受限
